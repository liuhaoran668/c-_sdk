# AGENTS.md

## Cursor Cloud specific instructions

### Project overview

LinkerHand C++ SDK — a C++17 library for controlling LinkerHand robotic hands via Linux SocketCAN. No external dependencies beyond pthreads and Linux kernel headers. See `CLAUDE.md` for full architecture and code style details.

### Build

```bash
cmake -S . -B build_local -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
cmake --build build_local
```

Output: `build_local/liblinkerhand_cpp_sdk.a` (static library).

### Lint / Static Analysis

- **cppcheck**: `cppcheck --enable=all --suppress=missingIncludeSystem --std=c++17 -I include/ src/`
- **clang-tidy**: `clang-tidy -p build_local src/*.cpp src/hand/l6/*.cpp --extra-arg="-std=c++17" --extra-arg="-isystem/usr/include/c++/13" --extra-arg="-isystem/usr/include/x86_64-linux-gnu/c++/13"`
  - Requires `build_local/compile_commands.json` (generated by the cmake configure step with `-DCMAKE_EXPORT_COMPILE_COMMANDS=ON`).
  - The `--extra-arg` flags are needed because clang 18 on Ubuntu 24.04 looks for GCC 14 headers but only GCC 13 is installed.

### Testing

No automated test suite exists in the repository. The test programs referenced in `CLAUDE.md` (`test_force_sensor`, `test_angle`) have empty definitions in `CMakeLists.txt`.

To compile and link a test against the library:
```bash
g++ -std=c++17 -I include/ your_test.cpp -L build_local -llinkerhand_cpp_sdk -lpthread -o your_test
```

### Runtime caveat

The library requires a physical CAN bus adapter and LinkerHand hardware. On cloud VMs without CAN hardware, constructing `L6` will throw `linkerhand::CANError` ("Failed to open CAN socket: Address family not supported by protocol"). This is expected behavior — the library compiles and links correctly; only runtime CAN I/O is blocked.

### Toolchain note

The default `c++` symlink on the VM may point to `clang++`, which fails to link against `libstdc++`. The update script switches it to `g++` via `update-alternatives`. If you see linker errors about `-lstdc++`, run: `sudo update-alternatives --set c++ /usr/bin/g++`.
