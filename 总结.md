### 1.2 核心组件职责

| 组件 | 职责 |
|------|------|
| **L6/L10/...（设备类）** | 统一入口，持有所有 Manager，管理共享生命周期 |
| **Manager** | 特定功能的数据采集与控制（角度、力传感器等） |
| **CANMessageDispatcher** | CAN 消息收发、订阅分发 |
| **Lifecycle** | 状态机管理，确保 close 后 API 行为一致 |
| **IterableQueue** | 线程安全的生产者-消费者队列，支持流式数据 |

## 2. 生命周期管理

### 2.1 设计目标

确保所有对外 API 在 `close()` 后具有一致的行为契约：
- **统一抛出 `StateError`**，而非未定义行为或不同类型的异常
- **阻塞操作立即唤醒**，不会等待超时
- **清理操作始终可执行**（如 `stop_streaming()`）

### 2.2 状态机设计

```
          ┌─────────────────────────────────────────┐
          │              Lifecycle                  │
          │  ┌───────┐              ┌────────┐     │
          │  │ Open  │─────────────▶│ Closed │     │
          │  └───────┘    close()   └────────┘     │
          │      │                       │         │
          │      ▼                       ▼         │
          │  允许操作                 拒绝操作      │
          │                         (StateError)   │
          └─────────────────────────────────────────┘
```

### 2.3 共享 Lifecycle 模式

```cpp
class L6 {
  std::shared_ptr<Lifecycle> lifecycle_;  // 共享状态

  AngleManager angle;        // 共享同一个 lifecycle_
  ForceSensorManager force;  // 共享同一个 lifecycle_
};

// Manager 在每个公开方法入口检查状态
void AngleManager::set_angles(const std::array<int, 6>& angles) {
  lifecycle_->ensure_open();  // 非 Open 状态抛 StateError
  impl_->set_angles(angles);
}
```

### 2.4 API 行为矩阵

| API 方法 | Open 状态 | Closed 状态 |
|----------|-----------|-------------|
| `set_angles()` | 正常执行 | 抛出 `StateError` |
| `get_*_blocking()` | 正常执行 | 立即抛出 `StateError` |
| `get_current_*()` | 正常执行 | 抛出 `StateError` |
| `stream()` | 正常执行 | 抛出 `StateError` |
| `stop_streaming()` | 正常执行 | **正常执行**（清理操作） |
| `close()` | 执行关闭 | 幂等，直接返回 |
| `is_closed()` | 返回 `false` | 返回 `true` |

## 3. 回调系统

### 3.1 串行回调架构

CAN 消息接收和回调处理在同一线程串行执行：

```
recv_thread（接收线程）：
┌────────────────────────────────────────────────────────┐
│  while (running) {                                      │
│    ① poll(10ms) 等待 CAN 消息                           │
│    ② read() 读取 CAN 帧                                 │
│    ③ for (sub : subscribers_copy) {                     │
│         if (sub->active.load()) {                       │
│           sub->callback(msg);  // 同步调用               │
│         }                                               │
│       }                                                 │
│  }                                                      │
└────────────────────────────────────────────────────────┘

特点：
- 单线程处理，无并发复杂度
- 回调快速执行（内存操作为主，无 I/O）
```

### 3.2 订阅者状态管理

```cpp
struct SubscriberState {
  std::size_t id = 0;
  Callback callback;
  std::atomic<bool> active{true};

  void deactivate() { active.store(false, std::memory_order_release); }
};
```

### 3.3 退订流程

退订设计简洁，依赖架构保证而非复杂同步机制：

```cpp
void CANMessageDispatcher::unsubscribe(std::size_t subscription_id) {
  std::lock_guard<std::mutex> lock(subscribers_mutex_);
  auto it = std::find_if(...);
  if (it == subscribers_.end()) return;

  (*it)->deactivate();   // 标记为非活跃，recv_loop 会跳过
  subscribers_.erase(it);
}
```

**架构约束**：安全性依赖于 `L6::close()` 的调用顺序——`dispatcher_.stop()` 会 join `recv_thread_`，确保在 Manager 析构前没有回调在执行。

## 4. 传感器多帧聚合

### 4.1 问题背景

力传感器数据量大（每个手指 72 字节），超过 CAN 帧的 8 字节限制：

```
单个手指的力传感器数据：72 字节
CAN 帧有效载荷：6 字节（8 字节中 2 字节用于协议头）
所需帧数：72 / 6 = 12 帧
```

### 4.2 帧格式

```
CAN 帧结构（8 字节）：
┌──────────┬──────────┬──────────────────────────────────┐
│ data[0]  │ data[1]  │ data[2..7]                       │
│ 命令前缀  │ 帧索引    │ 6 字节有效载荷                    │
│ (0xB1-B5)│ (高4位)  │                                  │
└──────────┴──────────┴──────────────────────────────────┘

命令前缀: 0xB1(拇指) 0xB2(食指) 0xB3(中指) 0xB4(无名指) 0xB5(小指)
帧索引: 0x00, 0x10, 0x20, ... 0xB0（对应 0-11 帧）
```

### 4.3 原地聚合 (FrameBatch)

```cpp
class FrameBatch {
  std::array<uint8_t, 72> values_{};
  std::array<bool, 12> received_{};
  std::size_t count_ = 0;

public:
  void add_frame(size_t idx, const uint8_t* payload) {
    if (!received_[idx]) {
      received_[idx] = true;
      ++count_;
    }
    std::copy_n(payload, 6, values_.begin() + idx * 6);
  }

  bool is_complete() const { return count_ == 12; }

  ForceSensorData assemble() const {
    ForceSensorData out{};
    out.values = values_;  // 直接拷贝
    out.timestamp = now_unix_seconds();
    return out;
  }

  void reset() {
    received_.fill(false);
    count_ = 0;
  }
};
```

**设计要点**:
- 原地修改避免 Python 式不可变拷贝开销
- `reset()` 复用对象，减少构造/析构
- 幂等帧处理：重复帧覆盖数据但不重复计数

### 4.4 数据流图

```
CAN 总线                         SingleForceSensorManager
    │                                     │
    ▼                                     ▼
┌─────────┐                        ┌─────────────┐
│ Frame 0 │───┐                    │ FrameBatch  │
│(0xB1,0x00)│  │                    │ (原地修改)   │
└─────────┘  │                     │             │
    ...      │  on_message()       │ count: 0→12 │
┌─────────┐  │                     └──────┬──────┘
│Frame 11 │──┘                            │ is_complete()
│(0xB1,0xB0)│                              ▼
└─────────┘                        ┌─────────────┐
                                   │  assemble() │
                                   └──────┬──────┘
                                          │
                                          ▼
                                   ┌─────────────┐
                                   │ForceSensorData│
                                   │ values[72]  │
                                   └─────────────┘
```

## 5. 线程安全设计

### 5.1 互斥锁保护

`send()` 和 `stop()` 共享 `socket_mutex_`：

```cpp
void CANMessageDispatcher::send(const CanMessage& msg) {
  std::lock_guard<std::mutex> lock(socket_mutex_);
  if (socket_fd_ < 0) {
    throw StateError("CAN dispatcher is stopped");
  }
  ::write(socket_fd_, &frame, sizeof(frame));
}
```

### 5.2 原子操作使用

```cpp
std::atomic<bool> running_{false};           // 控制 recv_loop 退出
std::atomic<bool> streaming_running_{false};  // 控制 streaming_loop 退出

// 内存序选择
running_.store(true, std::memory_order_release);   // 发布新状态
running_.load(std::memory_order_acquire);          // 获取最新状态
```

**注意**：`Lifecycle` 的状态通过 `std::mutex` 保护而非原子变量，因为状态变更需要同时通知订阅者，属于复合操作。

## 6. 资源管理与清理

### 6.1 RAII 与析构顺序

```cpp
class L6 {
  // 声明顺序决定析构顺序（逆序析构）
  std::shared_ptr<Lifecycle> lifecycle_;  // 最后析构
  CANMessageDispatcher dispatcher_;       // 倒数第二

  AngleManager angle;         // 依赖 dispatcher_，先析构
  ForceSensorManager force;   // 依赖 dispatcher_，先析构

  ~L6() {
    try {
      close();  // 显式清理，确保有序关闭
    } catch (...) {}
  }
};
```

### 6.2 关闭流程

```cpp
void L6::close() {
  if (!lifecycle_->close()) return;  // 幂等

  try {
    force_sensor.stop_streaming();
    angle.stop_streaming();
  } catch (...) {}

  try {
    dispatcher_.stop();
  } catch (...) {}
}
```

## 7. 数据流设计

### 7.1 阻塞式数据获取

```cpp
AngleData get_angles_blocking(std::chrono::milliseconds timeout) {
  auto waiter = std::make_shared<AngleWaiter>();
  {
    std::lock_guard lock(waiters_mutex);
    waiters.push_back(waiter);
  }

  if (!is_streaming) {
    send_sense_request();
  }

  std::unique_lock lock(waiter->mutex);
  bool ok = waiter->cv.wait_for(lock, timeout, [&] {
    return waiter->ready || lifecycle->state() != LifecycleState::Open;
  });

  if (ok && waiter->data.has_value()) return *waiter->data;

  lifecycle->ensure_open();
  throw TimeoutError("No data received within timeout");
}
```

### 7.2 并发多指阻塞查询

```cpp
AllFingersData get_data_blocking(std::chrono::milliseconds timeout) {
  auto deadline = steady_clock::now() + timeout;

  FingerArray<std::future<ForceSensorData>> futures;
  for (size_t i = 0; i < kFingerCount; ++i) {
    futures[i] = std::async(std::launch::async, [&] {
      return fingers[i]->get_data_blocking(timeout);
    });
  }

  AllFingersData out{};
  for (size_t i = 0; i < kFingerCount; ++i) {
    auto remaining = deadline - steady_clock::now();
    futures[i].wait_for(remaining);
    out.fingers[i] = futures[i].get();
  }
  return out;
}
```

### 7.3 流式数据采集

```
stream() 调用
     │
     ▼
┌─────────────────┐     ┌─────────────────┐
│ Streaming Thread│────▶│ IterableQueue   │
│ (定时发送请求)   │     │ (force_put)     │
└─────────────────┘     └─────────────────┘
                               │
                               ▼
                        用户迭代消费
                        for (data : queue)

stop_streaming() 或 close()
     │
     ▼
queue.close() → 迭代器结束
```

## 8. 添加新型号手示例

### 8.1 设计原则

> **核心原则：每个型号独立实现**
>
> - 每个型号一个独立文件夹
> - 每个型号一个设备类（如 `L6`、`L10`）
> - 文件夹内包含该型号所需的所有 Manager
> - 不使用继承或组合包装

### 8.2 目录结构

```
include/linkerhand/hand/
├── l6/      # L6 型号（独立完整）
├── l10/     # L10 型号（独立完整）
└── l20/     # L20 型号（独立完整）

src/hand/
├── l6/
├── l10/
└── l20/
```

### 8.3 示例

```cpp
// include/linkerhand/hand/l10/l10.hpp
namespace linkerhand::hand::l10 {

class L10 {
  std::shared_ptr<Lifecycle> lifecycle_;
  CANMessageDispatcher dispatcher_;
  uint32_t arbitration_id_;

public:
  L10(std::string side, std::string interface_name);
  ~L10();

  AngleManager angle;  // L10 自己的 AngleManager (10 DOF)

  void close();
  bool is_closed() const;
};

}  // namespace linkerhand::hand::l10
```

### 8.4 注册到统一入口

```cpp
// include/linkerhand/linkerhand.hpp
#include "linkerhand/hand/l6/l6.hpp"
#include "linkerhand/hand/l10/l10.hpp"

namespace linkerhand {
using L6 = hand::l6::L6;
using L10 = hand::l10::L10;
}
```

### 8.5 使用

```cpp
#include <linkerhand/linkerhand.hpp>
using namespace std::chrono_literals;

linkerhand::L6 hand_l6("left", "can0");
hand_l6.angle.set_angles({100, 100, 100, 100, 100, 100});
auto angles = hand_l6.angle.get_angles_blocking(100ms);
hand_l6.close();
```
